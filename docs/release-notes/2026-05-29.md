# Nova rev notice — 2026-05-29 (batch upload reliability)

**Subject: Batch upload now shows you what it found before pricing, and portfolio reports export reliably**

Audience: Fuelled team

This rev covers two issues Harsh hit on a batch upload today, in the same feature area (the multi-item batch tool on the Pricing page).

---

## What Harsh hit

> Was trying batch upload. It did not price anything.

The upload reported "14/14 priced" but every value came back $0, and the portfolio report wouldn't download.

## What was going on

Two separate problems, both now fixed:

1. **The spreadsheet wasn't a clean table.** It had the company name and a date on top, then the equipment lower down. The old uploader assumed row 1 was always the column header, so it grabbed the wrong column and fed Nova junk (a company name, a date, a header label) instead of equipment. Nova correctly couldn't value those — but the tool still counted them as "priced" at $0, which hid the problem.
2. **The portfolio report quietly failed.** Larger reports take about a minute to build, and the request was timing out before it finished — with no error shown, so it just looked like nothing happened.

## What we built

**1. A review step before pricing.** After you upload, Nova now shows you the line items it pulled out of the file, with a checkbox on each. You uncheck anything that isn't real equipment (a header row, a subtotal, a blank line), then click **Price**. Nothing gets priced — and nothing costs anything — until you've eyeballed the list. No more silently pricing garbage.

**2. Smarter reading of messy spreadsheets.** When a file isn't a clean one-row-per-item table, Nova now reads it with its document understanding (the same engine that reads P&IDs and emails) instead of a rigid column-reader. On Harsh's file it correctly skipped the company name, the date, and the header, and pulled out the three real units with make/model/year.

**3. An honest "priced" count.** Items without enough detail to value now show up as failures with a reason ("needs make/model, size, year, or condition") instead of being counted as priced at $0. The summary now reflects only items that actually got a valuation.

**4. Portfolio Report export fixed.** The .docx report now generates reliably, and shows a **"Generating… (~1 min)"** indicator while it works so you know it's running. If anything does go wrong, you'll now see an error instead of silence.

## How to use it

1. Go to **Pricing → Batch Upload** and drop in your `.xlsx`, `.csv`, or `.eml` as before.
2. Click **Upload & Extract Items**. Nova reads the file and shows you the items it found.
3. **Review the list.** Uncheck any rows that aren't equipment, then click **Price N selected items**.
4. When pricing finishes, use **Export XLSX** for the spreadsheet or **Portfolio Report** for the formatted .docx (give it about a minute — you'll see the "Generating…" label).

## What's NOT in this rev

- **No change to how individual items are priced** — same valuation engine, same comps, same methodology. This is only about getting the right items in and the results out.
- **No editing of extracted rows yet.** You can drop a row in review, but not rename or re-categorize it. If extraction gets a title wrong, fix it in the source file and re-upload. We can add inline editing later if it turns out to be needed.
- **Single-item pricing (the chat tool) is unchanged.**

## What's next
