# Nova rev notice — 2026-05-27

**Subject: Tier 2 pricing families now live — heater, treater, knockouts, dehydrator, meter runs**

Audience: Fuelled team

This rev brings 5 new equipment families through Tier 2 pricing. Each family gets its own ruleset — RCN brackets, depreciation curve, service factors — instead of being aliased to a generic curve. Net effect: when Nova prices an item in one of these families it'll produce a more defensible number with a per-family reasoning trail.

---

## What was asked for

> We need the pricing pass to actually be Tier 2 for the family. The current generic approach is leaving money on the table for these classes of equipment.

(Strategy doc: `docs/May 13 2026 - pricing strategy/Fuelled_Pricing_Strategy_2026-05-13.pdf`.)

## What we built

Five new family-specific pricing rulesets, each replacing the generic-curve fallback that was being used for these equipment classes. Per family:

- **Dehydrator** — TEG / mole-sieve / generic variant split, MMSCFD-scaled RCN brackets, dedicated curve flattened at mid-life (no longer aliased to treater)
- **Heater** — ANSI × MMBTU bracket logic, dedicated depreciation curve (no longer aliased to treater)
- **Treater** — electrostatic sour-bypass service factor, depreciation curve flattened at mid-life
- **Knockouts** — 4-way disambiguator (FWKO / Gas / Flare / Ambiguous) with per-sub-family pricing
- **Meter runs** — pipe × ANSI × meter-type bracket; LACT units flagged for review

Every priced row carries a methodology pointer telling you which family ruleset produced the number.

## What's new this rev

- **Dehydrator, heater, treater, knockouts, meter runs** now go through dedicated Tier 2 rulesets when you price them via the chat agent or the batch report path. Previously these classes shared a generic curve and produced flatter, less defensible numbers.
- **Knockout disambiguation** — if you upload an inventory that just says "knockout" without specifying type, Nova will flag it as ambiguous and ask before pricing. Cleaner than guessing wrong.
- **LACT meter runs get a review flag** rather than a silent price. Custody-transfer hardware has its own valuation logic that isn't well-served by the bracket curve.
- **Better error messages** in the pricing chat when something goes wrong. Previously a rate limit or model issue surfaced as "Something went wrong"; now you get a specific message (e.g. "Pricing model is rate-limited, retry in a moment" or "PDF too large, try a smaller file").

## How to use it

1. Open **Pricing Agent** as usual.
2. Price any item in these families — describe it or upload a spec sheet. Nothing changes in the input flow.
3. The result includes the family name in the methodology section, so you can tell at a glance which ruleset was used.
4. If a batch report has knockouts that didn't specify type, expect a flag rather than a number — that's the new behavior, not a regression.
