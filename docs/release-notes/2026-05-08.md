## What Mark asked for

A buy-side competitive tool that turns stale competitor inventory into a working acquisition list:

1. Detect competitor listings that have been sitting unusually long.
2. Treat that staleness as a buy-side signal.
3. Rank the best candidates so the team can pursue them.
4. Generate a Fuelled-ready draft packet for any candidate we want to chase.

## What we built

**Where to find it:** Nova → Competitive Intelligence → *Stale Acquisition Candidates*.

### Stale detection (category-relative)

Not every category goes stale at the same rate, so the bot uses different thresholds:

- **180 days** — compressors, engines, generators, VRUs, pump jacks
- **270 days** — separators, treaters, dehydrators, line heaters, scrubbers
- **365 days** — tanks, vessels, trailers, storage, miscellaneous heavy iron

Fuelled's own inventory is excluded everywhere — competitor-only by construction.

### Acquisition score (0–100)

Each candidate is ranked on five components so the top of the list is actually worth your time, not just old:

1. **Age pressure** (0–35) — how far past its category threshold the listing has gone.
2. **Negotiability** (0–20) — how far the asking price sits above the peer median (more room to negotiate).
3. **Liquidity** (0–20) — whether the category has enough live priced peers to support resale.
4. **Data quality** (0–15) — make / model / year / condition / hours / location completeness.
5. **Source quality** (0–10) — dealer / classified beats auction-style for buy-side.

Auction sources (Ritchie, IronPlanet, GovDeals, BidSpotter, AllSurplus, EnergyAuctions) still show in the feed but aren't auto-promotable — auctions are their own workflow.

### Promotion + acquisition queue

Click **Promote** on a candidate to move it into the internal acquisition queue. The queue is a durable, separate state store (so it survives deploys and is independent from the read-only listings feed). Each entry tracks status, assignee, notes, and a generated draft Fuelled listing packet you can pull into the listing flow when ready.

## What's new this week: seller contact info on the candidates

The biggest change: the Stale Targets table now shows a **SELLER** and **CONTACT** column next to every ranked candidate. When an old listing surfaces, you can see who to call without leaving the page.

This is powered by new data we just started ingesting:

### AllSurplus

- **Seller name + account type** (Commercial / Private) on every listing.
- **"Other assets" link** so you can see everything else that seller has live.
- **Event-level contact name / email / phone** parsed off the auction event page, where AllSurplus exposes it. Carried through to every candidate in that event.

### Kijiji (deployed 2026-05-06)

- **Dealer name + account type** (Dealer / Private) and dealer profile URL on every listing.
- Verified live on a known dealer — Global Power Systems, COMMERCIAL_50389.
- The next scheduled scrape (every 6h) keeps backfilling new and existing rows.

Both feed the same per-seller aggregation, so a single dealer running multiple consignments collapses correctly into one row instead of fragmenting.

## How to use it

1. Go to **Competitive Intelligence**.
2. Look at *Stale Acquisition Candidates*. Top of the list = highest score = best candidate to chase.
3. Use the **SELLER** and **CONTACT** columns to identify who to reach out to.
4. Click **Promote** on anything you want to pursue — it joins the acquisition queue with a draft listing packet ready.
5. Manage status / notes / assignment from the queue.

Feedback, missing data, or "this seller is wrong" — reply here or send to Curtis.
