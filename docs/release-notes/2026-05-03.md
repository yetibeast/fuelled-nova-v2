# Nova release notes — 2026-05-03

Audience: all Fuelled users.

## Pricing accuracy

- **Hours-inversion fix (Shawn's catch, 2026-04-28).** Adding a high-hour reading to an older unit was inflating the valuation instead of decreasing it. Hours can now only age a unit, never juvenate it. Repro: 2000 CAT G3306 / Sullair PCC20L at 61,000 hrs went from $116K → $52K, which lines up with comps. PR #3, regression tests added.

## New: see how Nova thinks

- **Methodology page** now shows a 5-stage visual pipeline (intake → comps → RCN → factors → output) instead of the old tile grid.
- **Worked example** under *Methodology → Example* walks through a generic compressor end-to-end so users can see exactly what each factor does to the number.

## Personalized for role

- Shawn (sales) and Shreya (content/marketing) now get role-tuned responses by default — same engine, framing tuned to job. Everyone else's experience is byte-for-byte unchanged.

## Bulk pricing

- **Spreadsheet uploads** now handle non-schema files via LLM fallback — vendor price lists and inventory exports work without manual schema mapping.
- **Pricing coverage widget** on the dashboard shows what fraction of a list Nova can price at high / medium / low confidence before committing to a run.

## Competitive intel

- New **stale acquisition workflow** flags listings that have been sitting and surfaces re-pricing candidates.

## Gold tables

- Editing now saves correctly (form was silently dropping `validation_status`).
- API errors surface in the modal instead of failing quietly.

## Known follow-ups

- Tightening RCN consistency across multi-turn conversations (math is right; LLM re-interprets specs mid-thread).
- Telemetry on rcn_engine fallback frequency.

## Internal-only changes (not in user note)

- Centralized `LOG_DIR` env var so pricing/feedback/reports history survives Railway redeploys.
- `.railwayignore` to keep `.next/node_modules` out of build context.
- Scraper INSERT now generates UUID explicitly (prod table lacked default).
- Admin user-create supplies UUID explicitly.
